How
  1. Without going into the specifics of the django project itself (as the commit history is all available using this link), the deployment was relatively simple.

  2. I developed this site on my desktop pc using Microsoft's "Remote - WSL" extension from visual studio code. My Raspberry PI's /var/www/jobiewebsite directory was set up in exactly the same way as my wsl virtual debian machine, other than using a different .env file, which contains the django secret key along with database credentials.

  3. I added my ssh key to the pi, forwarded the ports on my router for ssh and the port for my django site to be reached. I set up No-IP so that jobiewinser.ddns.net redirects to my non-static ip.

  4. I set up gunicorn and nginx to use http to safely deploy my website and also serve static/media files correctly.
Why

Put simply, it was a fun challenge to measure my progress as a full-stack developer and especially my DevOps skills. I wanted to deploy my own site as I would in a professional setting.

As a secondary objective, I wanted a tech demo to demonstrate my ability to rapidly create dynamic web systems that can be updated and changed by non-tech savvy users.

With the above in mind, the home page contains various technologies that I have used throughout my career which are contained in a PostgreSQL database and able to be edited and or appended quickly through the django admin panel. New entries are then available immediately on the homepage and filterable by Language, Project or Technology used.

The structure for the Projects, Languages and Technologies can be found at: models.py.
